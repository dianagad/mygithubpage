About the Project

Over the past few years, with the development of larger and more complex problems, the speed of computation and the management of complexity has become a basic problem in programming. So, Parallel Computing was introduced in order to develop a solution to optimize the time to completion by using multiple computing resources simultaneously especially to provide concurrency and save time or money.

Nowadays, the access to multicore computers makes it easy to improve the performance in a sequential programming language making the process of parallelization complicated. But there has been work on the design of parallel programs and rules reasoning about them have been developed [Parallel Program Design].

With these improvements and demands it is essential to make sure that parallel software works properly. This can be done with formal and mechanical verification, which can identify problems in early stages of the development and test both parallel algorithm and software used in embedded devices, multimedia and streaming applications and real-time systems like Air Traffic Control Systems, Networked Multimedia Systems and Command Control Systems.

The purpose of the project is to formalise the powerlist data structure proposed by Misra [Powerlist] in Isabelle in order to mechanically verify the correctness of the Fast Fourier Transform and possibly other algorithms based on the divide and conquer strategy and compare the resulting development to the provers RRL [An overview of Rewrite Rule Laboratory (RRL)], ACL2 [A Formalization of Powerlist Algebra in ACL2] and Coq [Programming and Reasonning with PowerLists in Coq], showing the benefits and the difficulties of the formalisation and reasoning.
